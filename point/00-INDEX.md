# Pointint Point Index

> [!info]
> **Last Updated:** 2026-03-27  
> **Point Name:** `point`
> **Current Focus:** Brand, MVP, Business, Obsidian note system

Pointint 옵시디언 볼트의 기본 원칙은 아래와 같다.

- 브랜드는 넓게, 첫 제품은 좁게
- 지금은 프론트보다 기능과 비즈니스 판단 기준을 쌓는다
- 아이디어는 먼저 `00-Inbox`에 받고, 정리된 내용만 핵심 폴더로 이동한다
- 최종 의사결정은 설계 문서와 구현 문서로 이어지게 만든다
- 내부 기준 문서는 `point/`, 외부 공유 문서는 `docs/`에 둔다

## Folder Map

### `00-Inbox`

정리되지 않은 생각을 가장 먼저 넣는 곳이다.

- 떠오른 아이디어
- 카피 문장
- 갑자기 생각난 기능
- 나중에 검토할 질문
- 회의 메모 초안

### `01-Core`

프로젝트의 가장 중심이 되는 비전, 로드맵, 타깃, 핵심 정의를 둔다.

### `02-Architecture`

시스템 구조, 데이터 구조, API 구조처럼 구현 기반을 설명하는 문서를 둔다.

### `03-Features`

기능 단위 문서를 쌓는 곳이다. 기능 정의, 사용자 흐름, 상태 전이, 세부 규칙을 담는다.

### `04-AI-System`

AI 프롬프트, 응답 구조, 모델 역할, 보조 AI/생성 AI 설계를 모은다.

### `05-Operations`

운영 정책, 비용, 지표, 배포, 세션 플로우를 둔다.

### `06-Implementation`

실제 구현 계획, 우선순위, 마일스톤을 정리한다.

- `plans/`는 활성 계획 문서와 내부 전략 기준본을 둔다

### `07-Brand`

Pointint의 철학, 카피, 내러티브, `모니테리어` 개념, 테마팩 세계관을 다루는 곳이다.

### `08-Business`

티어, 가격, 내부 재화, 판매 구조, 공식 콘텐츠, 마켓플레이스 전략을 정리한다.

### `09-Research`

경쟁사, 유저 인터뷰, 검증 가설, 관찰 메모를 모은다.

### `10-Journal`

세션 로그와 빠른 의사결정 기록을 남긴다.

### `11-Design`

비주얼 톤앤매너, 색상 팔레트, UI 방향, 브랜드 비주얼 가이드를 다루는 곳이다.

### `90-Archive`

더 이상 현재 판단 기준이 아닌 문서나 이전 버전을 보관한다.

### `98-Templates`

아이디어, 결정, 실험, 주간 리뷰 등 재사용할 노트 템플릿을 둔다.

### `99-Reference`

레퍼런스와 참고 자료를 모은다.

## Suggested Flow

`00-Inbox -> 07-Brand / 08-Business / 09-Research -> 01-Core / 03-Features -> 06-Implementation/plans -> 10-Journal -> 90-Archive`

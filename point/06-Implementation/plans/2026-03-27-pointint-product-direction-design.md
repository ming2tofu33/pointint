# Pointint Product Direction Design

- Date: 2026-03-27
- Status: Working draft
- Brand: Pointint

## 1. Summary

Pointint should be understood in two layers:

- as a focused product that starts with cursor creation
- as a broader brand for `모니테리어`

The immediate product promise is clear and narrow:

`Bring your own image -> turn it into a Windows cursor -> tune it -> simulate it -> download it.`

The long-term brand vision is broader than cursors alone. Pointint should become a platform for decorating and personalizing the digital desktop environment, starting with the smallest but most expressive desktop object: the cursor.

For now, strategic work should focus on product function and business structure, not frontend execution.

## 2. Brand Narrative

Pointint should lead with feeling and philosophy before feature explanation.

### Core brand stack

- Brand name: `Pointint`
- Main line: `Your Point, Your Tint.`
- Concept line: `for 모니테리어`

### Meaning of the name

`Point` should not be treated as only the mouse cursor.

It can mean:

- the point you select
- the point you notice
- the point you move toward
- your point of view
- the visual focus of your digital space

`Tint` means the mood, nuance, color, and personal atmosphere you layer onto that point.

This gives Pointint a much wider foundation than a cursor-only tool. The cursor is the first and most obvious point, but not the only one.

### Brand communication rule

Pointint should create a layered message:

1. philosophy first
2. concept second
3. product explanation third

Recommended order:

- `Your Point, Your Tint.`
- `Pointint for 모니테리어`
- then explain the current product, such as turning an image into a polished Windows cursor

This preserves expansion room while keeping the first product clear.

## 3. Product Thesis

Pointint is not just another cursor download site. It is a creation workflow.

Most existing competitors focus on one of these:

- distributing ready-made cursor packs
- browser-only cursor extensions
- older desktop editing tools for power users

Pointint focuses on the missing layer:

- image-based cursor creation for normal users
- visual hotspot control
- simulation before download
- trust and guidance for real Windows application

In short:

- Short-term identity: the easiest way to make your own Windows cursor from an image
- Long-term identity: a platform for `모니테리어`, beginning with cursors

## 4. Scope Strategy

Pointint should not start as a broad desktop customization product.

That would dilute positioning too early. The better strategy is:

1. win a narrow but memorable use case
2. prove that users want to personalize their desktop through Pointint
3. expand from cursor creation into adjacent `모니테리어` surfaces

The cursor workflow is the wedge product, not the final boundary of the brand.

## 5. MVP Definition

The MVP should exclude image generation.

The MVP goal is to validate whether people actually want the core creation workflow:

- upload a photo or image
- remove or refine the background
- crop and tune the cursor visual
- set the hotspot
- preview the cursor in realistic states
- download the result and apply it

### MVP feature set

- image upload
- basic preprocessing
- optional background removal
- basic manual cleanup
- resize and transform
- hotspot selection
- 3-state preview: `Normal`, `Text`, `Link`
- `.cur` export
- zip download
- Windows apply guide

### Explicit MVP exclusions

- text-to-image cursor generation
- image-to-image cursor generation
- full 17-role automatic generation
- animation-heavy creation flow
- broader `모니테리어` categories beyond cursors

## 6. Post-MVP AI Roadmap

AI should arrive in two layers.

### Phase A: AI-assisted editing

The first AI layer improves user-supplied images rather than generating brand-new ones.

Examples:

- smarter background removal
- edge cleanup
- automatic centering and cropping
- cursor readability enhancement
- silhouette cleanup for small sizes

This keeps the product grounded in the proven cursor workflow while improving quality.

### Phase B: AI generation entry point

After the cursor editor is proven, Pointint should add an AI generation entry point before the same editor.

The product should not create a separate AI product. Instead, the flow should be:

`Generate -> Edit -> Expand -> Simulate -> Download`

## 7. AI Generation Design

The AI generation feature should support three input modes in one product surface:

- text only
- image only
- reference image plus text

The experience should feel unified from the user's point of view. Internally these can be different pipelines, but externally the user should feel like they are simply telling Pointint what kind of cursor they want.

### First generation output

The first output should be one representative cursor draft, not a full pack.

Reason:

- easier to judge quality
- lower cost
- faster iteration
- more control before expanding into states

### Expansion behavior

If the representative cursor looks good, Pointint should then suggest auto-expanded variants for:

- `Normal`
- `Text`
- `Link`

The user should be able to:

- accept the suggested variants
- edit each state manually
- keep a mixed set of AI-suggested and manually adjusted states

This creates the right balance between AI speed and creative control.

## 8. Shared Editor Architecture

Pointint should have one main editor, with two starting paths:

- `Start with your image`
- `Start with AI`

Both paths should land in the same editor.

That editor should remain the center of the product. This keeps the experience coherent and prevents the AI layer from feeling detached from the real Pointint workflow.

### Shared editor responsibilities

- source asset display
- crop and transform
- hotspot control
- state switching
- preview and simulation
- export preparation

## 9. 모니테리어 Vision

The long-term concept behind Pointint is `모니테리어`.

Just as interior design shapes a room, `모니테리어` shapes the visual identity of a personal digital workspace. Pointint can become a product for users who want their monitor, desktop, and daily interface experience to feel more personal, expressive, and intentional.

### What Pointint means in that future

Pointint is not only a cursor tool. It is a brand for desktop personalization.

Future surfaces may include:

- cursor sets
- wallpaper generation and editing
- icon-related assets
- desktop interaction themes
- bundled personalization packs
- Pointint-made desktop pet applications

These are future directions, not current MVP commitments.

### Product role split inside the ecosystem

- user-made: cursor sets
- user-made: wallpapers and desktop backgrounds
- Pointint-made: desktop pet applications
- bundle layer: `모니테리어` theme packs that combine them

This split is strategically useful because not every object has to be user-generated. Pointint can be both a tool platform and a creator brand.

### Important strategic rule

Even with the broader `모니테리어` vision, Pointint should keep the first product promise extremely clear:

`Pointint helps you make your own cursor beautifully and easily.`

The broad vision should inform branding and roadmap, but it should not blur the initial product message.

## 10. Theme Pack Strategy

The real long-term business unit should not remain a single cursor file. It should evolve into the `theme pack`.

A `모니테리어` theme pack can eventually contain:

- cursor set
- wallpaper
- mood metadata or style label
- desktop pet, when relevant
- future decorative assets

This is strategically important because theme packs are easier to:

- merchandise
- share
- collect
- feature editorially
- price as premium content

### Strong supporting ideas

- theme pack builder
- desktop or monitor mock preview
- cursor-wallpaper harmony recommendations
- official seasonal collections
- artist or IP collaboration packs
- community gallery with remix and save flows

## 11. Positioning Statement

Recommended near-term positioning:

`Pointint is the easiest way to turn your image into a polished Windows cursor.`

Recommended concept positioning:

`Pointint is for 모니테리어.`

Recommended future positioning:

`Pointint is a platform for 모니테리어, starting with cursors.`

## 12. Business Perspective

Pointint should be treated as a business with multiple monetization layers over time.

### Primary business ladder

1. make creation useful
2. make personalization coherent
3. make theme packs desirable
4. make premium content collectible
5. make the ecosystem sellable

### Revenue surfaces

- Pro subscription for advanced export and workflow power
- AI credits for generation-heavy features
- premium `모니테리어` theme pack sales
- Pointint-made desktop pet sales
- official drops and seasonal collections
- marketplace commissions
- artist, brand, or IP collaborations
- future licensing or B2B customization opportunities

### Important monetization principle

In the early phase, the product should earn trust before it tries to monetize the full vision.

That means:

- first win on cursor creation
- then expand into wallpaper and theme pack value
- then introduce high-emotion premium objects such as pets and official drops

## 13. Strategic Decisions Already Made

- Final product name is `Pointint`
- Main brand line is `Your Point, Your Tint.`
- Concept framing is `for 모니테리어`
- MVP should focus on upload/edit/simulate/download
- AI image generation should not be included in MVP
- AI should be added after MVP in a staged way
- AI generation should sit in front of the existing editor, not as a separate product
- AI generation should support `text only`, `image only`, and `image + text`
- AI should first generate one representative cursor draft
- Auto expansion should begin with `Normal`, `Text`, and `Link`
- Users should still be able to edit each expanded state manually
- Pointint should be treated as the first wedge into a broader `모니테리어` platform
- wallpaper generation and editing is a likely next user-made surface
- desktop pets should be Pointint-made downloadable Windows applications
- theme packs should become the long-term packaging unit

## 14. Open Questions

These are not blockers for the MVP, but they matter for the next design pass:

- what should be the first paid upgrade trigger after the MVP proves value
- how strongly `for 모니테리어` should appear in the first landing page
- whether the first premium content should be official theme packs or desktop pets
- how far auto-expansion should go before introducing the full 17-role system

# Product Differentiation

> [!info]
> **Status:** Active  
> **Last Updated:** 2026-03-27  
> **Purpose:** Pointint의 제품 차별점을 기능, 워크플로우, 경쟁 비교 관점에서 정리한다.

## 한 줄

`Pointint는 이미지를 실제로 쓸 수 있는 Windows 커서로 가장 쉽게 바꾸는 웹 기반 제작 워크플로우다.`

## Executive Summary

제품 관점에서 Pointint의 차별점은 아주 선명하다.

1. **이미지 한 장으로 시작한다**
2. **Windows 실사용을 전제로 만든다**
3. **Hotspot과 시뮬레이션을 결과물만큼 중요하게 다룬다**
4. **비회원도 충분히 써보게 하고, 이후 저장/확장/AI로 이어진다**
5. **커서에서 끝나지 않고 나중에 모니테리어 테마팩으로 확장할 수 있다**

즉 Pointint는 `갤러리`, `브라우저 확장`, `파워유저 에디터` 사이의 빈 공간을 채우는 제품이다.

---

## 1. 무엇과 다른가

### 1.1 다운로드 라이브러리와의 차이

다운로드 라이브러리는 아래 흐름에 강하다.

`Browse -> Download -> Install`

Pointint는 아래 흐름에 강하다.

`Upload -> Edit -> Hotspot -> Simulate -> Download`

차이는 명확하다.

- 라이브러리는 “고르는 경험”
- Pointint는 “만드는 경험”

### 1.2 브라우저 확장과의 차이

브라우저 확장은 빠르고 가볍지만, 브라우저 정책과 영역 제한이 있다.  
Pointint는 처음부터 **실제 Windows 커서**를 기준으로 설계한다.

차이점:

- 브라우저 내부 페이지 제약이 없음
- OS 사용 맥락 중심
- 3종, 17종 세트 개념으로 확장 가능
- 나중에 테마팩과 연결 가능

### 1.3 파워유저 에디터와의 차이

Cursor.cc나 RealWorld Cursor Editor는 만들 수 있지만 어렵다.

Pointint는 아래 쪽에 가깝다.

- 이미지 기반 진입
- 설치 없는 웹 경험
- 감각적인 결과물 우선
- 기능보다 흐름 우선

즉 Pointint는 “제작” 카테고리에 있으면서도, `전문 툴`이 아니라 `대중 제작기`에 가깝다.

## 2. 제품 차별화 포인트

### 2.1 Image-first input

Pointint는 픽셀 노가다에서 시작하지 않는다.  
사용자가 갖고 있는 이미지에서 시작한다.

이 차이는 매우 크다.

- 진입 장벽이 낮다
- 비디자이너도 바로 이해할 수 있다
- 감정적으로 “내 것”이 된다

### 2.2 Windows-first output

Pointint는 “귀여운 커서 구경”보다 “실제로 쓸 수 있는 커서”를 목표로 한다.

이 차이를 만드는 요소:

- `.cur` 결과물
- Hotspot 지정
- 3종 / 17종 세트 구조
- 적용 가이드

### 2.3 Hotspot and simulation first

많은 플레이어가 예쁜 비주얼에 치우친다.  
Pointint는 실사용 품질을 전면에 둔다.

- 어디를 눌러야 하는지
- 텍스트 영역에서 어떻게 보이는지
- 링크 위에서 어떻게 느껴지는지

즉, Pointint는 `aesthetic`와 `usability`를 같이 다룬다.

### 2.4 Strong free experience

Pointint의 제품 구조는 차별점이기도 하다.

- 비회원도 업로드 기반 제작을 무제한으로 경험
- `Normal / Text / Link` 3종까지 다운로드 가능
- 로그인 이후 저장, 전체 세트, AI 확장

이 구조는 “가입해야 써볼 수 있는 툴”보다 훨씬 강한 첫인상을 만든다.

### 2.5 AI as extension, not gimmick

Pointint에서 AI는 처음부터 제품 정체성을 흐리지 않는다.

- MVP: 업로드 기반 제작
- 다음: AI 보조
- 이후: AI 생성

그리고 생성도 별도 제품이 아니라 기존 흐름 앞단에 붙는다.

`Generate -> Edit -> Expand -> Simulate -> Download`

이 구조 덕분에 AI가 장난감처럼 보이지 않는다.

## 4. Pointint가 실제로 이겨야 하는 포인트

### 4.1 “더 많은 커서”가 아니라 “더 쉬운 나만의 커서”

기존 시장은 풍부한 콘텐츠가 이미 있다.  
Pointint는 양으로 이기지 않는다.

이겨야 하는 질문:

`남의 커서를 고를래, 아니면 내 이미지를 내가 쓰는 커서로 바꿀래?`

### 4.2 “예쁜 장난감”이 아니라 “예쁘고 실제로 쓰는 것”

Pointint는 귀여운 비주얼만으로 끝나면 안 된다.

- 예뻐야 한다
- 눌러야 한다
- 보여야 한다
- 적용되어야 한다

이 실사용 축이 차별점의 핵심이다.

### 4.3 “한 번 쓰고 끝”이 아니라 “확장 가능한 시작점”

처음에는 커서 3종이지만, 그다음은 아래로 이어질 수 있다.

- 17종 전체 세트
- 저장과 재수정
- AI 보조/생성
- 커뮤니티 공유
- 테마팩
- 배경화면
- 데스크톱 펫

Pointint는 처음 기능은 작아도, 제품 구조는 작지 않다.

---

## 5. 약점과 우회 전략

### 5.1 커서는 원래 바이럴이 약하다

커서는 내 화면에서만 보이는 오브젝트라 공유성이 약하다.

이 약점을 우회하려면:

- 시뮬레이션 장면 자체를 예쁘게 보여주기
- 제작 완료 쇼케이스 이미지 자동 생성
- “before / after” 제작 과정 노출
- 나중에 커서 + 배경을 묶는 테마 맥락 만들기

### 5.2 커서는 니치다

그래서 Pointint는 초기 포지셔닝은 커서에 집중하되, 브랜드 프레임은 더 넓게 가져가야 한다.

즉:

- 제품 설명은 좁게
- 브랜드 설명은 넓게

---

## 6. 최종 정리

### Verdict

제품 관점에서 Pointint의 차별점은 아래 한 문장으로 정리된다.

`Pointint는 이미지 기반 진입, Windows 실사용 중심, Hotspot과 시뮬레이션을 전면에 둔 웹 기반 커서 제작 워크플로우다.`

이 한 문장 덕분에 Pointint는:

- 라이브러리보다 더 개인적이고
- 브라우저 확장보다 더 실용적이며
- 파워툴보다 더 쉽게 쓸 수 있다

그리고 그 위에 AI와 모니테리어 확장까지 얹을 수 있다.

# Pointint Conversion & SEO Optimization

## Context

### 핵심 전환 퍼널

```
방문 → Guest 제작 → 시뮬레이션 → 다운로드 → [가입] → [유료 전환]
```

### 전환 포인트별 트리거

| 전환 | 트리거 | 심리 |
|---|---|---|
| 방문 → 제작 시작 | 이미지 업로드의 낮은 허들 | "이거 한번 해볼까?" |
| 제작 → 시뮬레이션 | 결과물 확인 욕구 | "어떻게 보일까?" |
| 시뮬레이션 → 다운로드 | 실사용감 확인 | "이거 쓸 만하네" |
| Guest → Free | 17종 전체 + 저장 | "다른 커서도 바꾸고 싶다" |
| Free → Lite | AI 보조/생성 | "AI로 더 예쁘게 만들고 싶다" |

### V1 성공 기준

- 비회원이 이미지 한 장으로 커서를 만들고 다운로드까지 끊김 없이 완료
- 시뮬레이션 후 다운로드 비율이 의미 있게 높음
- 사용자가 실제로 Windows에 적용
- 로그인 전환이 "저장하고 싶다"는 자연스러운 동기에서 발생

## Workflow

### 1. 시스템 스킬 체이닝

**CRO 요청:** `page-cro` 스킬 호출 → Pointint 퍼널 맥락 적용
**SEO 요청:** `seo-audit` 스킬 호출 → 커서 제작 키워드 맥락 적용

### 2. CRO 분석 프레임

Pointint 전환 최적화 시 반드시 점검할 항목:

**A. 첫 화면 (랜딩)**
- [ ] 업로드 CTA가 스크롤 없이 보이는가?
- [ ] "Your Point, Your Tint." 톤이 유지되는가?
- [ ] 제작 결과물 예시가 보이는가?
- [ ] 설명이 짧고 여백이 충분한가?

**B. 제작 흐름**
- [ ] 업로드 → 편집 → 시뮬레이션 전환이 끊기지 않는가?
- [ ] 각 단계에서 다음 행동이 명확한가?
- [ ] 로딩/대기 시간에 시각적 피드백이 있는가?

**C. 전환 포인트**
- [ ] Guest → Free 전환 제안이 자연스러운 타이밍인가?
- [ ] 강요 없이 가치 기반으로 제안하는가?
- [ ] 전환 후 즉시 체감되는 혜택이 있는가?

# 소셜 공유 메타

## 원칙

Pointint 링크가 SNS에 공유됐을 때, 클릭하고 싶은 미리보기가 떠야 한다. 특히 쇼케이스 개별 작품은 해당 커서가 직접 보여야 바이럴이 된다.

---

## Phase 1 — 랜딩 공유

| 항목 | 값 |
|---|---|
| og:title | Pointint — Your Point, Your Tint. |
| og:description | 이미지를 나만의 Windows 커서로 만드는 가장 쉬운 방법 |
| og:image | Pointint 대표 브랜드 이미지 (정적) |
| og:type | website |
| twitter:card | summary_large_image |
| twitter:title | 동일 |
| twitter:description | 동일 |
| twitter:image | 동일 |

### 체크리스트

- [ ] og:title, og:description, og:image 설정
- [ ] twitter:card summary_large_image 설정
- [ ] 한/영 각각 메타 태그 (다국어 대응)
- [ ] 대표 OG 이미지 제작 (1200×630px 권장)
- [ ] 카카오톡 공유 미리보기 확인
- [ ] 트위터/X 공유 미리보기 확인
- [ ] Discord/Slack 공유 미리보기 확인

---

## Phase 3b — 쇼케이스 개별 작품 공유

| 항목 | 값 |
|---|---|
| og:title | {유저명}의 커서 — Pointint |
| og:description | Pointint에서 만든 커서를 구경해보세요 |
| og:image | **해당 커서의 미리보기 이미지 (동적 생성)** |
| og:type | website |
| twitter:card | summary_large_image |

### 동적 OG 이미지

- 커서 작품별로 OG 이미지를 동적으로 생성해야 한다
- 배경 + 커서 미리보기 + Pointint 브랜딩을 합성한 이미지
- Next.js의 `opengraph-image` 또는 별도 이미지 생성 엔드포인트로 구현
- 이미지 크기: 1200×630px

### 체크리스트

- [ ] 동적 OG 이미지 생성 로직 구현
- [ ] 커서 미리보기 + 브랜딩 합성 템플릿 디자인
- [ ] 유저명 / 커서 이름 동적 삽입
- [ ] .ani 커서의 경우 대표 프레임 1장 사용
- [ ] 캐싱 (같은 작품은 OG 이미지 재생성 방지)
- [ ] 주요 SNS 미리보기 테스트 (카카오톡, 트위터, Discord)

---

## 바이럴 시나리오

```
유저가 커서를 만든다
→ 쇼케이스에 공유한다
→ SNS에 링크를 올린다
→ 친구가 미리보기에서 커서를 본다
→ "나도 만들어볼까?" 클릭
→ Pointint 유입
```

이 루프가 돌려면 **OG 이미지에서 커서가 매력적으로 보여야** 한다.

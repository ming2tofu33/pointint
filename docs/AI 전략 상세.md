# AI 전략 상세

## 핵심 원칙

- AI는 Pointint의 제작 워크플로우를 더 강하게 만드는 기능이다
- AI가 Pointint의 정체성을 대체하지 않는다
- Pointint의 핵심은 어디까지나 `제작 -> 검증 -> 적용` 흐름이다
- 생성 AI도 반드시 기존 편집기로 이어져야 한다

---

## 1. MVP에서의 AI

MVP에서 AI는 “와, AI다”로 느껴지는 기능이 아니라 **기본 품질 레이어**에 가깝다.

### MVP에서 허용되는 AI/자동 처리

- 배경 정리 또는 배경 제거
- 포맷 정리
- 기본 품질 정규화

### 중요한 해석

- 사용자는 AI를 쓰고 있다고 느끼지 않아도 된다
- Pointint는 “업로드하면 알아서 커서로 만들 준비를 해준다” 정도로 느껴지면 된다
- 전처리가 완벽하지 않아도 편집 흐름은 계속 이어져야 한다

### MVP에서 하지 않는 것

- 텍스트 기반 커서 생성
- 이미지 기반 새 커서 생성
- 상태별 자동 스타일 확장

---

## 2. Phase A: AI 보조 편집

MVP의 제작 흐름이 검증된 뒤, 가장 먼저 붙이기 좋은 AI는 생성이 아니라 보조다.

### 기능 후보

| 기능 | 설명 | 효과 |
|---|---|---|
| 배경 제거 고도화 | 더 정밀한 엣지 처리 | 결과물 품질 향상 |
| 가장자리 보정 | 잔여 픽셀 정리 | 실루엣 정리 |
| 자동 크롭/정렬 | 오브젝트 영역 자동 인식 | 편집 시간 단축 |
| 가독성 개선 | 작은 크기에서 더 잘 보이게 보정 | 실사용 품질 향상 |
| 실루엣 정리 | 불필요한 디테일 축소 | 커서다운 형태 강화 |

### 위치

이 기능들은 편집기 안에서 “선택적으로 쓰는 보조 도구”로 붙는 것이 맞다.

---

## 3. Phase B: AI 생성 진입점

AI 보조가 안정된 뒤에는 생성 AI를 붙일 수 있다.

중요한 점은 생성 AI가 별도 제품이 아니라, **기존 편집기 앞단에 붙는 진입점**이어야 한다는 것이다.

### 권장 흐름

`Generate -> Edit -> Expand -> Simulate -> Download`

### 입력 방식

- 텍스트만
- 이미지만
- 참고 이미지 + 텍스트

### 첫 결과물

- 대표 커서 1개
- 사용자가 마음에 들면 `Normal / Text / Link` 3종 변형안 추천
- 상태별 직접 수정 가능

### 원칙

- 생성 결과물은 반드시 편집기로 이어져야 한다
- “AI가 다 해준다”가 아니라 “AI가 시작점을 만든다”가 맞다

---

## 4. AI와 티어의 관계

### Guest / Free

- 기본 전처리 레이어만 체감
- 사용자가 직접 호출하는 AI 기능은 없음 또는 극히 제한적

### Lite

- AI 보조 편집 접근
- AI 생성은 `하루 3회` 수준의 명확한 사용 한도

### Pro

- 더 높은 AI 한도 또는 월간 크레딧
- 더 좋은 생성/관리 경험

### 운영 방식

- 화면에는 “하루 3회”처럼 단순하게 보이게
- 내부적으로는 credit 구조를 쓰는 것이 유연하다

예:

- 보정 1회 = 1 credit
- 생성 1회 = 2 credits

---

## 5. 모델 선택 기준

구체적인 모델명보다, 현재는 아래 기준이 더 중요하다.

| 기준 | 설명 |
|---|---|
| 배경 정리 품질 | 작은 크기에서도 가장자리가 깔끔한가 |
| 처리 속도 | 사용자가 기다린다고 느끼지 않는가 |
| 비용 | Guest 사용량까지 감당 가능한가 |
| 작은 오브젝트 적합성 | 커서 같은 작은 결과물에 유리한가 |
| 확장성 | 보조 AI에서 생성 AI로 이어지기 쉬운가 |

---

## 6. 한 줄 정리

Pointint에서 AI는 처음엔 “보이지 않는 품질 레이어”로 시작하고, 나중에는 “생성 진입점”으로 확장된다. 하지만 어떤 단계에서도 Pointint의 본질은 AI 자체가 아니라 제작 워크플로우다.

# 운영 정책

## 1. 사용량 정책

### Guest (비회원)

- 제작 횟수 제한 없음
- 기본 전처리, 편집, Hotspot, 시뮬레이션, 3종 다운로드 가능
- 정상 사용자는 제한을 느끼면 안 된다

### 남용 방어

일반 사용자가 아니라 비정상 요청만 막는 구조가 적절하다.

| 방어 수단 | 설명 |
|---|---|
| Rate limiting | 비정상 속도의 연속 요청 차단 |
| 파일 크기 제한 | 업로드 최대 크기 상한 |
| 해상도 제한 | 처리 가능한 최대 해상도 상한 |
| 동일 이미지 캐싱 | 같은 이미지 반복 처리 방지 |
| 봇 탐지 | 필요 시 CAPTCHA 또는 행동 기반 탐지 |

### 운영 원칙

- 정상 사용자 경험을 해치지 않는다
- 방어는 UX보다 기술 레이어에서 처리한다
- “무제한”은 방치가 아니라, 조용한 방어 위에서 성립한다

---

## 2. 파일 관리

### 업로드 파일

- 비회원: 임시 저장 후 자동 만료
- 로그인 사용자: 프로젝트 단위로 저장, 재수정 가능

### 결과물 파일

- MVP 결과물은 `.cur` 및 `.ani`
- 비회원 결과물은 장기 보관하지 않는다
- 로그인 사용자는 프로젝트와 결과물 이력을 관리할 수 있다

### 원칙

- 비회원 데이터는 최소 보관
- 저장과 재수정은 로그인 가치로 연결
- 스토리지 비용과 사용자 편의를 함께 본다

---

## 3. 비용 방어

Pointint는 Guest 경험을 강하게 여는 대신, 비용 통제를 반드시 설계해야 한다.

### 통제 방식

| 방법 | 설명 |
|---|---|
| 해상도 정규화 | 처리 전 적정 크기로 리사이즈 |
| 결과 캐싱 | 동일 이미지 해시 기반 재처리 방지 |
| 처리 레이어 분리 | 무거운 처리와 가벼운 처리를 구분 |
| 모델 선택 최적화 | 품질 대비 비용이 적절한 방식 선택 |
| 가능한 클라이언트 처리 | 서버 부담을 낮출 수 있는 경우 우선 검토 |

### 장기 비용 구조

- MVP: 전처리 / 배경 정리 비용이 주된 변동 비용
- Phase A: AI 보조 기능 추가로 비용 증가
- Phase B: AI 생성 비용은 Lite/Pro 및 credit 구조와 직접 연결

## 4. 신뢰 & 안전

### 기본 신뢰 경로

- 기본 적용 방식은 `zip + .cur + 적용 가이드`
- 사용자가 Windows 설정에서 직접 적용
- 자동 설치 성격의 기능은 후순위 또는 고급 옵션

### 왜 수동 적용이 기본인가

- 사용자가 통제감을 느낀다
- “내 PC를 앱이 바꾼다”는 불안을 줄인다
- 원복이 쉽다는 점을 같이 보여줄 수 있다

### 원복 가이드

- 모든 다운로드에는 원복 방법을 함께 안내
- 적용 방법만큼 원복 방법도 쉽게 보여준다

### 공유 정책

- 공개 공유는 명시적 동의 후에만 가능
- 비회원 결과물은 공유 대상이 아님
- 마켓플레이스 오픈 시 별도 공유/판매 동의 흐름 설계

## 5. 개인정보

### 비회원

- 개인 식별 정보 수집 최소화
- 업로드 이미지는 처리 후 만료
- 장기 보관하지 않음

### 회원

- 이메일 등 최소 정보만 수집
- 프로젝트 데이터는 계정에 귀속
- 계정 삭제 시 관련 데이터 삭제

---

## 6. 운영 모니터링

### 핵심 지표

| 지표 | 목적 |
|---|---|
| 제작 완료율 | 업로드 -> 다운로드까지 끊기지 않는지 |
| 시뮬레이션 -> 다운로드 전환율 | 시뮬레이션 가치 검증 |
| Guest -> 로그인 전환율 | 무료 경험이 가입으로 이어지는지 |
| 처리 시간 | 대기 경험 관리 |
| 서버 비용 / 제작 건수 | 건당 비용 추적 |
| 남용 차단 비율 | 방어 시스템 효과 확인 |

### 원칙

- 사용자 경험과 비용을 같이 본다
- 지표가 나빠질 때는 먼저 기술적 최적화를 검토한다

## 7. 콘텐츠 / 법적 정책

NSFW, 저작권/IP, 결과물 소유권 정책은 추후 별도 문서로 분리하는 것이 적절하다.

현재 기준 핵심 방향만 정리하면 아래와 같다.

- MVP에서는 업로드 이미지 장기 저장을 최소화한다
- 공개 공유와 판매는 명시적 동의 이후에만 가능하다
- 결과물은 원칙적으로 사용자 중심으로 본다
- AI 생성물 소유권과 판매 가능성은 Phase B 이전에 별도 법률 검토가 필요하다

# 모바일 대응 정책

## 원칙

커서 제작은 데스크톱 전용이다. 모바일에서는 Pointint를 알게 되고, 구경하고, 가이드를 보는 것까지만 지원한다.

## 화면별 대응

| 화면 | 모바일 대응 | Phase | 비고 |
|---|---|---|---|
| 랜딩 | **반응형** | Phase 1 | SNS/검색에서 모바일 유입의 첫 인상 |
| 편집기 | **데스크톱 전용** | Phase 1 | 모바일 접속 시 "데스크톱에서 만들어보세요" 안내 |
| 시뮬레이션 | **데스크톱 전용** | Phase 1 | 마우스 커서가 없으면 체감 불가 |
| 다운로드/적용 가이드 | **반응형** | Phase 1 | PC 옆에 폰 놓고 따라하는 시나리오 |
| 쇼케이스/갤러리 | **반응형** | Phase 3b | SNS 공유 링크로 모바일에서 구경 |
| 유저 프로필 | **반응형** | Phase 3b | 다른 사람 작품 모바일에서 감상 |
| 마켓플레이스 | **반응형** | Phase 5a | 구경/구매는 모바일에서도 가능 |

## 모바일에서 편집기 접속 시

- 편집 화면 대신 안내 메시지를 보여준다
- "Pointint 커서 제작은 데스크톱에서 가장 잘 동작합니다"
- 랜딩 또는 쇼케이스로 유도

## 근거

- Hotspot 지정은 32px 캔버스에서 픽셀 단위 정밀도가 필요 — 터치로는 사실상 불가능
- 시뮬레이션은 마우스 커서가 있어야 의미 있음
- .cur/.ani는 Windows 전용 — 모바일에서 다운받아도 쓸 곳이 없음
- 편집기 모바일 대응 공수 대비 실사용이 없음

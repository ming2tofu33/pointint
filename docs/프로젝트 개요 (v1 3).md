# 문서 A — Pointint Project Overview (2026-03-27 refresh)

## 0) Executive Summary

**Main line**: `Your Point, Your Tint.`

**Concept line**: `Pointint for 모니테리어`

**Pointint**는 모니테리어를 위한 브랜드이자 제작 플랫폼이다. 다만 첫 제품은 넓게 시작하지 않고, **이미지를 나만의 Windows 커서로 만드는 가장 쉬운 워크플로우**에 집중한다.

현재 Pointint의 핵심 제품 정의는 아래와 같다.

`이미지 업로드 -> 편집 -> Hotspot 지정 -> 시뮬레이션 -> 다운로드`

즉 Pointint는 커서 다운로드 사이트가 아니라, **내가 원하는 디지털 오브젝트를 직접 만들고 검증하는 제작 파이프라인**이다.

### 현재 기준 핵심 원칙

- **브랜드는 넓게, 첫 제품은 좁게**: Pointint는 장기적으로 모니테리어 브랜드지만, 초반 제품은 커서 제작에 집중한다.
- **MVP는 커서 제작 흐름 검증**: 업로드 기반 커서 제작이 실제로 다운로드와 적용까지 이어지는지 증명하는 것이 우선이다.
- **무료 경험은 강하게**: 비회원도 충분히 만들고 써볼 수 있어야 한다.
- **AI는 MVP 이후 단계적 도입**: 먼저 업로드 흐름을 증명하고, 이후 AI 보조 -> AI 생성 순서로 붙인다.
- **장기 상품 단위는 테마팩**: 커서, 배경화면, 펫 같은 오브젝트를 하나의 모니테리어 패키지로 묶는 방향을 지향한다.

## 1) Brand Thesis

### 1.1 `Pointint`의 의미

`Point`는 단순히 커서를 뜻하지 않는다.

- 내가 선택하는 지점
- 내가 주목하는 포인트
- 내가 움직이는 방향
- 내 디지털 공간의 시선 중심
- 나의 관점

`Tint` 역시 단순한 색을 넘어 아래를 포함한다.

- 분위기
- 무드
- 뉘앙스
- 취향
- 감정적인 톤

즉 Pointint는 디지털 공간의 포인트에 사용자만의 분위기를 입히는 브랜드다.

### 1.2 왜 커서로 시작하는가

커서는 모니터 위에서 가장 작지만 가장 자주 움직이고, 가장 즉각적으로 개인 취향을 드러낼 수 있는 오브젝트다.

초기 제품으로서의 장점은 아래와 같다.

- 시도 장벽이 낮다
- 결과 체감이 빠르다
- 개성이 잘 드러난다
- 편집, Hotspot, 시뮬레이션, 다운로드라는 Pointint다운 흐름을 만들기 좋다

따라서 커서는 Pointint의 첫 번째 진입점으로 적합하다.

---

## 2) Problem

Pointint가 풀고 싶은 문제는 단순히 “커서 파일을 만든다”가 아니다.

더 본질적으로는 아래와 같은 문제를 푼다.

- 디지털 공간을 내 취향대로 꾸미는 과정이 생각보다 불편하다
- 기존 커서 생태계는 다운로드 사이트, 설치형 편집기, 브라우저 확장으로 파편화되어 있다
- 이미지가 있어도 실제로 예쁘고 쓸 만한 커서로 만들기 어렵다
- Hotspot, 크기, 배경 대비, 알파 처리 문제로 결과물이 불편해지기 쉽다
- 적용 전에 느낌과 품질을 확인하기 어렵다

Pointint는 이 문제를 **제작 워크플로우**로 해결한다.

---

## 3) Product Positioning

Pointint는 아래 셋 중 세 번째에 해당한다.

- 다운로드 사이트
- 단순 편집기
- 제작 워크플로우

Pointint는 명확하게 **제작 워크플로우**다.

한 줄로 정리하면 아래와 같다.

`Pointint는 커서를 시작으로 모니테리어를 만드는 제작 플랫폼이다.`

### 초반 제품 설명

`이미지를 나만의 Windows 커서로 가장 쉽게 바꾸는 방법`

### 장기 브랜드 설명

`커서에서 시작해 배경화면, 테마팩, 데스크톱 펫까지 확장하는 모니테리어 플랫폼`

---

## 4) MVP Definition

### 4.1 MVP의 핵심 질문

MVP는 아래 질문에 답해야 한다.

`사람들은 Pointint에서 이미지를 커서로 만들고, 실제로 다운로드해서 적용할 만큼 이 흐름을 가치 있게 느끼는가?`

### 4.2 MVP에 반드시 포함할 것

- 이미지 업로드
- 기본 전처리
- 배경 제거 또는 최소 편집
- 커서용 위치/크기 조정
- Hotspot 지정
- 3종 상태 시뮬레이션
  - `Normal`
  - `Text`
  - `Link`
- `.cur` 다운로드
- zip 패키징
- Windows 적용 가이드

### 4.3 MVP에서 제외할 것

아래 항목은 중요하지만 MVP에는 넣지 않는다.

- AI 생성
- AI 보조 고도화
- 17종 전체 자동 생성
- 테마팩
- 바탕화면 생성/편집
- 데스크톱 펫
- 마켓플레이스
- 내부 재화

### 4.4 권장 사용자 흐름

1. 이미지 업로드
2. 배경 정리 및 기본 편집
3. 커서 위치/크기 조정
4. Hotspot 지정
5. `Normal / Text / Link` 3종 시뮬레이션
6. 다운로드
7. 적용 가이드 확인

## 5) Usage Structure

### 5.1 Guest (비회원)

비회원도 Pointint의 핵심 가치를 충분히 체험할 수 있어야 한다.

제공 방향:

- 업로드 기반 제작 무제한
- 편집
- Hotspot 지정
- 시뮬레이션
- `Normal / Text / Link` 3종 세트 다운로드

제한:

- 프로젝트 저장 없음
- 재수정 없음
- AI 없음

### 5.2 로그인 이후 가치

로그인 이후 첫 번째 가치는 아래 두 가지다.

- 프로젝트 저장
- 더 완성된 결과물로의 확장

현재 논의 기준으로, 로그인 사용자는 아래 가치에 접근하는 구조가 자연스럽다.

- 프로젝트 저장 및 재수정
- 17종 전체 커서팩 다운로드 또는 확장 접근
- 이후 AI 기능 접근 기반

핵심은 **가입을 막으로 쓰지 않고, 더 깊은 개인화로 가는 확장 보상**으로 쓰는 것이다.

## 6) AI Roadmap

AI는 Pointint의 핵심 차별점이 될 수 있지만, **MVP 이후**에 붙여야 한다.

### Phase A: AI-assisted editing

먼저 도입할 AI는 생성보다 보조가 적합하다.

- 배경 제거 고도화
- 가장자리 보정
- 자동 크롭
- 작은 크기용 가독성 개선
- 실루엣 정리

### Phase B: AI generation entry point

그 다음에 생성 AI를 붙인다.

중요한 점은 생성 AI가 별도 제품이 아니라, **기존 편집기 앞단에 붙는 진입점**이어야 한다는 것이다.

권장 흐름:

`Generate -> Edit -> Expand -> Simulate -> Download`

### 생성 AI의 첫 결과물

- 첫 결과물은 `대표 커서 1개`
- 마음에 들면 `Normal / Text / Link` 3종 변형안 자동 추천
- 사용자는 상태별로 직접 수정 가능

지원 방향:

- 텍스트만
- 이미지만
- 참고 이미지 + 텍스트

## 7) 모니테리어 Expansion

Pointint의 상위 개념은 `모니테리어`다.

장기 확장은 아래 순서가 자연스럽다.

1. 커서
2. 정적인 바탕화면 이미지 생성/편집
3. Pointint가 직접 제작해 배포하는 데스크톱 펫 앱
4. 이를 묶는 모니테리어 테마팩

즉 Pointint는 무작정 넓어지는 제품이 아니라, 모니터 위의 오브젝트를 차례대로 확보하는 브랜드가 된다.

## 8) Business Direction

### 8.1 제품 구조

- **첫 번째 제품 축**: 유저 제작형 커서
- **다음 확장 축**: 유저 제작형 바탕화면
- **브랜드 시그니처 축**: Pointint 제공형 데스크톱 펫
- **장기 상품 단위**: 커서/배경/펫을 묶는 테마팩

### 8.2 수익 구조 방향

장기적으로는 아래 구조가 가장 자연스럽다.

- Guest: 비회원 체험
- Free: 로그인 회원 레이어
- Lite: 저가 유료층
- Pro: 고급 유저/크리에이터층
- Premium content: 공식 테마팩, 펫 팩
- Marketplace: 크리에이터 판매 수수료

### 8.3 내부 재화 방향

내부 재화는 초기에는 `AI 사용`과 연결하고, 중기 이후에는 `공식 콘텐츠 구매`까지 연결하는 하이브리드 구조가 적합하다.

한 줄로 정리하면 아래와 같다.

- AI는 재화의 바닥 수요를 만든다
- 콘텐츠는 재화의 감정적 가치를 만든다

---

## 9) Trust / Safety / Operations

### 9.1 기본 신뢰 경로

- 기본 경로는 **zip + .cur + 적용 가이드**
- 사용자가 Windows 설정에서 직접 적용하도록 안내
- 자동 설치 성격의 기능은 후순위 또는 고급 옵션으로 분리

### 9.2 운영 원칙

- 업로드 파일은 임시 저장 후 만료
- 공개 공유는 명시적 동의 이후
- 원복 가이드를 제공해 심리적 부담을 낮춤
- 비용 방어는 해상도 제한, 캐싱, 동일 이미지 재처리 방지로 설계

# 문서 B — Pointint Appendix

> 본 문서는 핵심 개요에서 다루지 않은 세부 스펙과 운영 항목을 위한 부록이다.

## A) Core Features 상세

### A1. Upload & Preprocess

- 포맷: PNG, JPG, WebP
- 크기 제한: 운영 상황에 따라 조정
- 배경 제거 옵션
  - 초기: 빠른 구현 우선
  - 이후: 자체 처리 또는 혼합 처리 가능
- 최소 수동 보정
  - 브러시/지우개
  - 가장자리 간단 보정

### A2. Cursor Editor

- Hotspot 드래그 지정
- Resize: 16–128px
- Transform: Flip, Rotate(90° step)
- 멀티사이즈 팩은 이후 확장 기준으로 관리

### A3. Preview / Simulator

- v1: `Normal`, `Text`, `Link`
- 이후: Windows 17 roles 전체
- 배경: Light/Dark
- hotspot 표시 토글, safe area 가이드(옵션)

### A4. Export / Download

- `.cur` 생성
- zip 패키징
- 적용 가이드
- 원복 가이드

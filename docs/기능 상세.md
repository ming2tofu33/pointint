# 기능 상세

## 1. 현재 기준 제품 범위

Pointint의 MVP는 **정적 커서(.cur)와 애니메이션 커서(.ani)** 제작을 모두 포함한다.

### 정적 커서 흐름

`이미지 업로드 -> 기본 전처리 -> 편집 -> Hotspot 지정 -> 3종 시뮬레이션 -> 다운로드`

### 애니메이션 커서 흐름

`이미지 여러 장 업로드 또는 GIF 업로드 -> 프레임 편집 (순서, 속도) -> 미리보기 -> 다운로드`

즉, 지금 단계의 Pointint는 “내 이미지를 실제로 쓸 수 있는 커서로 만드는 경험”을 가장 잘 만드는 데 집중한다.

---

## 2. MVP 파이프라인

### 지원 대상

- 정적 이미지 기반 커서 제작 (.cur)
- 애니메이션 커서 제작 (.ani) — 심플하게, 이후 고도화
- 출력 대상: `.cur` / `.ani`
- 시뮬레이션 대상: `Normal / Text / Link`

### MVP에서 제외

- 17종 전체 자동 생성
- AI 생성 진입점
- 마켓플레이스 연동
- 테마팩 묶음 제작
- 프레임별 개별 편집 (.ani 고도화)

---

## 3. 이미지 업로드 & 기본 전처리

### 업로드

- 지원 포맷: PNG, JPG, WebP, GIF
- 정적 이미지는 “한 장만 올리면 된다”고 느껴야 한다
- GIF는 애니메이션 커서(.ani) 제작 흐름으로 진입
- 내부 포맷 정리는 Pointint가 처리한다

### 기본 전처리

- 필요 시 배경 정리 또는 배경 제거
- 커서로 쓰기 어렵지 않도록 기본 정렬/정규화 처리
- 사용자는 파일 포맷이나 투명 배경 처리 과정을 의식하지 않아야 한다

### 원칙

- 전처리는 “AI 기능”처럼 보이기보다 기본 품질 레이어로 느껴져야 한다
- 전처리가 완벽하지 않아도 사용자가 편집 단계로 자연스럽게 넘어갈 수 있어야 한다

---

## 4. 커서 편집기

### MVP 조작 범위

| 조작 | 설명 |
|---|---|
| 위치 조정 | 커서 프레임 안에서 이미지를 이동 |
| 크기 조정 | 커서 규격에 맞춰 리사이즈 |
| 회전/뒤집기 | 최소한의 방향 보정 |
| Hotspot 지정 | 클릭 지점을 드래그로 설정 |

### MVP에서 의도적으로 빼는 것

- 브러시/지우개 중심의 이미지 편집
- 포토샵 수준의 색 보정
- 과한 효과 필터

### 핵심 원칙

- Pointint는 이미지 편집 앱이 아니라 커서 제작 도구다
- 편집기는 “커서로 만드는 데 필요한 최소 조작”만 제공해야 한다
- 가장 중요한 조작은 Hotspot과 실제 사용감이다

---

## 4-B. 애니메이션 편집기 (.ani)

### 진입 경로

- 이미지 여러 장 업로드: 각 이미지가 하나의 프레임
- GIF 업로드: 자동으로 프레임 분리

### MVP 조작 범위

| 조작 | 설명 |
|---|---|
| 프레임 순서 | 드래그로 변경 |
| 프레임 속도 | 전체 또는 개별 재생 속도 조절 |
| 프레임 추가/삭제 | 프레임 단위 관리 |
| 애니메이션 미리보기 | 실시간 재생 확인 |

### 이후 고도화

- 프레임별 개별 편집 (위치, 크기, Hotspot)
- 프레임별 배경 제거
- 프레임 간 보간/전환 효과

### 핵심 원칙

- MVP에서는 프레임 구성과 속도 조절에 집중
- 프레임별 세부 편집은 이후에 추가

---

## 5. 시뮬레이션

### MVP 시뮬레이션 범위

- `Normal`
- `Text`
- `Link`

### 시뮬레이션 방식

두 층으로 나누는 것이 적절하다.

**빠른 비교용 미리보기**

- 3종 상태를 한눈에 비교
- 빠르게 결과 품질을 확인

**실사용 감각용 인터랙티브 영역**

- 텍스트 위에 올리면 `Text`
- 링크 위에 올리면 `Link`
- 다운로드 전에 “실제로 쓰면 어떤지” 체감

### 배경

- Light / Dark 전환 지원

### .ani 시뮬레이션

- 애니메이션 커서는 시뮬레이션에서 재생 상태를 확인할 수 있다
- 속도감과 자연스러움을 다운로드 전에 검증

### 핵심 원칙

- 시뮬레이션은 단순 썸네일이 아니라 Pointint의 핵심 차별점이다
- 다운로드 전에 사용자가 결과를 확신할 수 있어야 한다

---

## 6. 다운로드 & 적용 가이드

### Guest 기본 다운로드

- `Normal / Text / Link` 3종 세트 (.cur 또는 .ani)
- 개별 다운로드 또는 세트 zip 패키지
- 적용 가이드 동반

### 로그인 이후 확장 가치

- 프로젝트 저장
- 재수정
- 17종 전체 커서팩 다운로드 또는 확장 접근

### 적용 가이드

- Windows 설정에서 수동 적용하는 기본 경로를 안내
- “어떻게 적용하는지”와 함께 “어떻게 원복하는지”도 보여준다

### 핵심 원칙

- 다운로드가 끝이 아니라 적용까지가 Pointint의 경험이다
- 적용 가이드는 기능이 아니라 제품 신뢰의 일부다

---

## 7. Guest와 로그인의 경계

### Guest

- 업로드
- 기본 전처리
- 편집
- Hotspot
- 시뮬레이션
- 3종 다운로드

### 로그인

- 프로젝트 저장
- 재수정
- 더 완성된 결과 접근
- 이후 AI 기능과 연동될 기반

### 원칙

- 핵심 제작 경험은 Guest에게도 충분히 열어둔다
- 가입은 막이 아니라 확장 보상이어야 한다
